## 2. Phase별 상세 평가 및 개선 제안

### Phase 0: 실제 예제 분석 (10분)

**현재 상태:**
- 예제 선택 (2분) + 4단계 구조 분석 (5분) + 동료 공유 (3분)

**문제점:**
1. **2분에 예제 선택은 너무 짧음**: 학생들이 4개 예제를 읽고 선택하기 어려움
2. **5분에 4단계 구조 분석은 충분함**: 하지만 구체적 가이드 필요
3. **3분 동료 공유는 의미 있음**: 하지만 더 구조화 필요

**개선 제안:**

**Magdalena Skipper 제안:**
- 예제 선택 시간을 3분으로 늘리고, 사전에 예제를 읽어오도록 과제에 포함
- Research integrity 관점에서 "이 예제들이 실제 논문인 이유" 명시

**Francesca Cesari 제안:**
- 예제 선택을 수업 전 과제로 이동 (사전 읽기)
- 수업 시간에는 바로 분석 시작 (시간 절약)
- 분석 가이드를 더 구체적으로 (체크리스트 제공)

**Victoria Aranda 제안:**
- Translational medicine 관점에서 예제 1개 추가 제안
- Nature editorial process와 연결: "이 논문들이 Nature에 제출되었을 때 어떻게 평가되었는가?"

**구체적 수정안:**
```
### Phase 0: 실제 예제 분석 (8분) [10분 → 8분으로 조정]

**사전 준비 (과제에 포함):**
- `examples_introduction_top_tier_patterns.md` 읽기
- 자신의 연구와 유사한 Gap 패턴 1개 선택 (선택 이유 메모)

**Activity 0: 탑티어 Introduction 패턴 분석**
- 선택한 예제의 4단계 구조 분석 (5분)
  - [체크리스트 제공]
  - Established Knowledge: 어떤 consensus가 제시되는가? (1분)
  - Emerging Challenges: 어떤 모순/한계가 제시되는가? (1분)
  - Critical Gap: Gap statement는 어떻게 표현되는가? (1.5분)
  - Research Opportunity: 어떤 방법/관점이 제시되는가? (1.5분)
- 동료와 선택한 예제 및 분석 결과 공유 (3분)
  - [구조화된 공유: 각자 1분씩 발표]
```

## 3. 과제 및 평가 기준 개선 제안

### 과제 평가

**현재 과제:**
- 총 6개 항목, 약 2,600 words

**문제점:**
1. **분량이 과도함**: 대학원생에게 부담
2. **항목 4 (탑티어 예제 비교 분석)는 중복**: Phase 0에서 이미 수행
3. **항목 5 (AI 활용 과정)는 유용함**: 하지만 더 구체적 가이드 필요

**개선 제안:**

**Magdalena Skipper 제안:**
- Research integrity 관점에서 "AI 활용 윤리" 항목 추가
- 과제 분량을 2,000 words로 줄임

**Francesca Cesari 제안:**
- 항목 4 (탑티어 예제 비교 분석)를 항목 1에 통합
- 항목 5 (AI 활용 과정)를 더 구체적으로 (프롬프트별 효과성 평가)

**Victoria Aranda 제안:**
- Nature editorial perspective 추가: "이 gap이 Nature에 제출될 때 필요한 추가 자료는?"

**구체적 수정안:**
```
### 과제 (다음 주까지)

### "My Research Gap - Validated & Justified"

**제출물:**

1. **Gap Discovery Documentation (600 words)** [800 → 600으로 감소]
   - **Landscape mapping 결과**
     - Consensus 3가지
     - Debates/contradictions 3가지
     - Unexplored areas 2가지
   - **Critical review analysis**
     - 논문 3편 이상 분석 [5편 → 3편으로 감소]
     - 각 논문의 한계점 (4가지 카테고리)
     - 공통 한계점에서 도출한 gap
   - **Gap validation 과정**
     - Google Scholar 검색 키워드 및 결과
     - 확인한 논문 리스트 (유사 연구 있었는가?)
     - AI 검증 결과

2. **Gap Statement (150-200 words)** [유지]

3. **Gap-driven 문헌 리뷰 구조 (400 words)** [500 → 400으로 감소]
   - Established knowledge (consensus)
   - Emerging challenges (contradictions/limitations)
   - Critical gap (what we don't know)
   - Research opportunity (why now)

4. **탑티어 예제 비교 분석 (200 words)** [300 → 200으로 감소, Phase 0 결과 활용]
   - Phase 0에서 선택한 예제의 4단계 구조 분석 요약
   - 자신의 Introduction 초안과 비교
   - 패턴 적용 방안

5. **AI 활용 과정 (400 words)** [500 → 400으로 감소]
   - 사용한 프롬프트 레시피 **5개 이상** (각 프롬프트별 효과성 평가)
   - 각 단계에서 AI가 어떻게 도움이 됐는지
   - AI의 한계 및 인간 판단이 필요했던 지점
   - False gap을 걸러낸 과정
   - **AI 활용 윤리**: AI 출력을 어떻게 검증하고 인용했는가?

6. **Peer Review 반영 (200 words)** [300 → 200으로 감소]
   - 받은 피드백 요약 (4가지 평가 항목별)
   - 각 피드백을 어떻게 반영했는지

**총 분량: 약 1,950 words** [2,600 → 1,950으로 감소]
```
